# 浅色主题：工作台化规则

日期：2026-03-29

## 一句话

**壳层退后，工作区前置，艺术性通过工作区构图获得。** 后续浅色主题迭代必须遵守这条。

## 问题背景

旧的浅色主题层级是反的：shell 背景和 chrome 比内容区更抢眼，卡片接近纯白没有层级，用户第一眼看到的是气氛而不是数据。

## 层级规则

从外到内，视觉权重递增：

1. **Shell background** — 只营造轻微氛围，低饱和低对比，不承担信息层级
2. **App chrome**（侧边栏等） — 可以有体积感但不能是主角
3. **Function region**（页面工作区） — 用户应该先识别"这页在操作什么"
4. **Primary card** — 比 shell 更重、比 function region 更聚焦，不能是近白色纸片
5. **Sub-card / detail block** — 必须与主卡形成可见的第二层级，用 tint/渐变/弱边框

## 结构做法

- **无全局顶栏**：原顶栏只承载品牌文案，浅色下容易把视觉重心拉回壳层。桌面端状态收进 sidebar，移动端用浮层。除非顶栏有新的强功能，不要恢复。
- **页面自己的 toolbar 是入口**：统计 chip、筛选、操作按钮形成自然入口区。不靠大标题或抽象口号撑层级。
- **Board stage**：机器页和任务页先有一个工作台面（board stage），卡片安放在控制台里而不是浮在背景纸上。
- **历史页走 ledger**：Archive Ledger → Filter View → 记录表格，三层不互相抢。
- **弹窗双区结构**：信息密度高的弹窗（如 TaskCreateModal）靠区域职责清晰获得可读性，不靠换背景色。

## 禁止事项

- 不要让 shell 或 header 成为全页最显眼的区域
- 不要让 card / function region / modal body 接近纯白
- 不要回到"标题 + 按钮 + 大块空白"的页面骨架
- 不要为了"更有艺术性"直接提高整体饱和度 — 先看是不是构图问题

## 后续精修顺序

1. 小字号、kicker、chip、filter bar 的字重与对比
2. 设置页和次级弹窗与主页面语言一致
3. 最后才改机器卡、任务卡、状态胶囊的微观质感
